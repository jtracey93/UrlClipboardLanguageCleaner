URL Clipboard Language Cleaner

Settings

General

When enabled, language/locale segments are automatically removed from URLs as you copy them.

Locale pattern

The regex used to detect locale/language path segments. Applied case-insensitively. Must match the leading / of the segment and use a lookahead for the trailing /.
Default: /[a-z]{2}(?:-[a-z]{2,4})?(?=/)

Excluded codes

Two-letter path segments listed here will not be removed because they are common English words that may appear legitimately in URLs (e.g. /us/, /in/, /at/). Note: codes with a region subtag (e.g. /no-nb/) are always removed regardless of this list. Enter one code per line, lowercase.
